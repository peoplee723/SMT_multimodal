# SMT_multimodal
SMT 공정 불량 및 설비 이상 탐지 프로젝트

AI Hub의 SMT 제조 데이터를 활용하여 제품 이미지 기반 불량을 탐지하고, 설비 센서 데이터를 분석하여 이상 상태를 확인하는 스마트 제조 AI 프로젝트입니다.
팀 프로젝트 전체 코드 중 본인이 담당한 객체탐지, kafka 통신 및 MongoDB저장 기능을 정리하였습니다.



### 1. 프로젝트 개요
SMT 공정은 PCB 기판에 전자 부품을 실장하고 납땜하는 제조 과정으로, 사전 공정(PR) 단계에서 발생한 납 도포 불량을 조기에 발견하지 못하면 이후 리플로우 공정 이후의 재작업·폐기 비용 증가로 이어질 수 있습니다. 또한 기존 검사 장비의 오탐은 작업자의 반복 검수를 유발하여 생산 라인의 처리 효율을 낮추는 문제가 있습니다. 이를 해결하기 위해 본 프로젝트에서 이미지 데이터와 설비 센서 데이터를 결합하여 SMT 공정의 불량을 탐지하고, 불량 발생 원인 후보를 분석해 대시보드로 제공하는 멀티모달 AI 기반 품질 관리 시스템을 개발하였습니다. 




### 2. 담당 역할
저는 프로젝트에서 다음 기능을 개발하였습니다.

- 데이터 전처리 (YOLO 학습에 맞게 데이터셋 가공)
- YOLO기반 SMT불량 객체 탐지 모델 학습 및 추론 구형
- Kafka기반 데이터 연동 구조 설계 및 구현
- Docker 기반 모델 실행 및 매포 파이프라인 구성



### 3. 구현 범위
#### 3.1 SMT 공정 불량 객체 탐지
   YOLO11n 모델을 활용하여 SMT공정에서 불량의 위치와 유형을 탐지하는 모델을 개발하였습니다.  
      주요기능
   - 이미지 입력
   - 불량 유형 분류
   - 불량 객체 Bounding Box 추출
   - 탐지 신뢰도 출력
   - 탐지 결과 이미지 저장
   
#### 3.2 Kafka 통신을 통한 실시간 이미지 처리 및 DB저장
   Kafka 통신과 MongoDB를 활용하여 실시간으로 이미지를 처리하고 결과를 DB에 저장하는 파이프라인을 개발하였습니다.  
      \t주요기능
   - 이미지 데이터 수신
   - 실시간 모델 예측
   - 예측 결과 DB저장
   
#### 3.3 Docker Compose를 활용한 멀티 컨테이너 애플리케이션 통합 실행 및 관리
   Docker Compose를 활용하여 프론트엔드, 백엔드, AI 모델 서버 등 여러 컨테이너를 하나의 설정 파일로 구성하고 일괄 실행·관리하였습니다.
